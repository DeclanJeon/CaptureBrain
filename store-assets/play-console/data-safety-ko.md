# Play Console 데이터 보안 답변 초안 - 캡처브레인

주의: 이 문서는 Play Console 입력을 돕기 위한 초안입니다. 실제 앱 동작과 배포 빌드 기준으로 최종 확인해야 합니다.

## 데이터 수집/공유 개요

현재 코드 기준:

- 앱은 Google Drive 연결을 위해 Google 계정 이메일을 처리합니다.
- 앱은 사용자의 스크린샷 이미지를 기기에서 읽고 OCR/분류/Markdown 생성을 수행합니다.
- 앱은 사용자가 연결한 Google Drive에 스크린샷 이미지와 Markdown/JSON 메타데이터를 업로드합니다.
- 현재 OCR은 Google ML Kit 온디바이스 라이브러리 사용으로 보이며, 별도 서버 업로드는 코드에서 확인되지 않았습니다.

## Play Console에서 민감하게 볼 항목

### 개인 정보
- 이메일 주소: Google 계정 연결 상태 및 Drive 업로드 계정 식별에 사용
- 목적: 앱 기능

### 사진 및 동영상
- 사진: 스크린샷 이미지 접근/처리/업로드
- 목적: 앱 기능

### 파일 및 문서
- Markdown/JSON 기록 생성 및 Drive 업로드
- 목적: 앱 기능

### 앱 활동/앱 정보
- 로컬 처리 상태, 큐 상태, 업로드 상태
- 목적: 앱 내부 기능

## 보안 관행

- 전송 중 데이터 암호화: Google Drive API는 HTTPS 사용
- 사용자가 데이터 삭제 요청 가능: 앱 삭제, Drive 파일 직접 삭제, Google 계정 권한 해제
- 데이터 수집 목적: 앱 기능 제공
- 광고 목적 사용: 아니오
- 제3자 판매: 아니오

## 권한 설명 문구

사진/이미지 접근 권한:
캡처브레인은 사용자가 찍은 스크린샷을 감지하고 자동으로 분류하기 위해 이미지 접근 권한을 사용합니다.

Google Drive 권한:
캡처브레인은 사용자가 연결한 Google Drive에 스크린샷 원본과 Markdown 기록을 저장하기 위해 Drive 권한을 사용합니다.

## 출시 전 확인 필요

- 외부 OCR/LLM/API로 이미지 또는 텍스트를 전송하는 코드가 있는지 최종 확인
- 서버 로그/분석 도구/Firebase Analytics 추가 여부 확인
- Crashlytics, 광고 SDK, 분석 SDK가 들어가면 이 답변을 반드시 갱신
