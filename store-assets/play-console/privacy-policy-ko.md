# 개인정보처리방침 초안 - 캡처브레인

시행일: 출시일 입력 필요

캡처브레인은 사용자의 스크린샷을 자동으로 정리하고 Google Drive에 저장하기 위해 필요한 최소한의 정보만 처리합니다.

## 1. 처리하는 정보

- 사용자의 Google 계정 이메일: Google Drive 연결 상태 표시 및 업로드 계정 식별
- Google Drive 접근 권한 토큰: 사용자가 허용한 범위에서 Drive 업로드 수행
- 사용자가 입력한 저장 폴더명
- 기기에 저장된 스크린샷 이미지
- 스크린샷에서 추출한 텍스트/OCR 결과
- 자동 분류 결과, 요약, Markdown 기록
- 업로드 대상 Google Drive 폴더/파일 ID

## 2. 이용 목적

- 스크린샷 감지
- 스크린샷 내용 분석 및 카테고리 분류
- Markdown 기록 생성
- 사용자의 Google Drive에 스크린샷 이미지와 Markdown 파일 저장
- 앱 내 최근 처리 상태 표시

## 3. 저장 위치

- 앱 설정과 처리 상태는 사용자의 기기 내부 저장소에 저장됩니다.
- 업로드된 파일은 사용자가 연결한 Google Drive에 저장됩니다.
- 현재 코드 기준 Google Drive 권한은 `drive.file` 범위를 사용하며, 앱이 생성하거나 사용자가 허용한 파일 작업에 필요한 범위로 제한합니다.

## 4. 제3자 제공 및 외부 처리

- Google Drive 저장 기능을 위해 Google Drive API를 사용합니다.
- 현재 코드 기준 OCR은 Google ML Kit 온디바이스 라이브러리를 사용합니다.
- 향후 외부 AI/LLM API로 스크린샷 내용 또는 OCR 텍스트를 전송하는 기능을 추가할 경우, 개인정보처리방침과 Play Console 데이터 보안 답변을 먼저 갱신해야 합니다.

## 5. 권한 사용

- 사진/이미지 접근 권한: 기기에 저장된 스크린샷을 감지하고 처리하기 위해 사용합니다.
- 인터넷/네트워크 권한: Google Drive 연결 및 업로드를 위해 사용합니다.
- 부팅 완료 수신 권한: 사용자가 허용한 자동 처리 흐름을 유지하기 위해 사용합니다.

## 6. 데이터 삭제 방법

- 앱 삭제: 기기에 저장된 앱 설정과 로컬 처리 데이터가 삭제됩니다.
- Google Drive에서 직접 삭제: 앱이 업로드한 스크린샷/Markdown 파일은 사용자의 Google Drive에서 직접 삭제할 수 있습니다.
- Google 계정 권한 해제: Google 계정 보안 설정에서 캡처브레인의 Drive 접근 권한을 해제할 수 있습니다.
